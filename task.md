Платформа автосервиса Mercedes — 4 слоя
1. 🌐 Публичный сайт
Премиальный дизайн, услуги, модели, цены, онлайн-запись (реальный календарь, не форма), отзывы, блог, команда, контакты.
2. 👤 Личный кабинет клиента
Мои авто, история обслуживания, статус ремонта онлайн, согласование сметы, счета, напоминания о ТО.
3. ⚙️ Админ-панель
Календарь записей, заказ-наряды (канбан), CRM клиентов, склад запчастей, цифровой осмотр авто, сметы и счета, аналитика, управление контентом сайта.

Стек: Next.js + PostgreSQL + Яндекс Cloud + ЮКасса + SMSC.ru
Фазы: Сайт + базовая админка → Личный кабинет → Склад и аналитика → Лояльность и магазин

4. 🛒 Магазин запчастей
Каталог с поиском по модели / артикулу / названию, фильтры (OEM / аналог, в наличии / под заказ), карточка запчасти с фото и совместимостью, корзина, оформление заказа, оплата онлайн (ЮКасса), история заказов в личном кабинете.
В админке: управление каталогом, остатки, цены, заказы покупателей, поставщики.

4.1 🔗 Публикация на внешних площадках
Из карточки запчасти в админке — одна кнопка, и объявление улетает на:

Авито (через Avito API)
Дром (через XML-фид)
Юла (через API)
Существующий сайт запчастей amggarage77.ru — синхронизация каталога

Что автоматически подтягивается: фото, название, описание, цена, артикул, совместимость с моделями. Редактировать вручную под каждую площадку не нужно.
В админке: статус публикации по каждой площадке, кнопка снять с публикации, статистика просмотров объявлений.
+ Zzap.ru
Один из главных авторынков в России для запчастей — обязательно. Интеграция через Zzap XML-фид (стандартный формат, который они принимают). Запчасти из каталога автоматически попадают в листинг Zzap с ценой, наличием и обновлением остатков в реальном времени.

Итого площадки для публикации: Авито, Дром, Юла, Zzap, собственный сайт запчастей.

Одна карточка в админке — публикация везде одновременно. Остатки и цены обновляются автоматически на всех площадках при изменении в системе.

Splus закрывает операционку хорошо. Строить своё с нуля — значит потратить год на то, что уже работает. Разумнее интегрироваться через их API и строить поверх то, чего у них нет. Это быстрее, дешевле и практичнее.

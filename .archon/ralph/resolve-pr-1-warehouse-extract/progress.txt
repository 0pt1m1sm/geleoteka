## Codebase Patterns

### PR branch lives in a sibling worktree
- **Where**: `~/.archon/workspaces/claude-dev/Geleoteka/worktrees/archon/task-archon-architect-1781275339869`
- **Pattern**: PR #1's head branch `archon/task-archon-architect-1781275339869` is already checked out in a sibling worktree. Make story-scoped edits there, commit, and push directly to that branch. Do NOT try to push from the ralph worktree (different base; would corrupt history).
- **Example**: `cd <sibling-worktree> && <edit file> && git add <file> && git commit -m ... && git push origin HEAD:archon/task-archon-architect-1781275339869`

### prisma generate requires DATABASE_URL to be set
- **Where**: `prisma.config.ts:14`
- **Pattern**: `prisma.config.ts` calls `env("DATABASE_URL")`. The resolver throws on missing/empty even for schema-only commands like `prisma generate`. Any non-empty string satisfies it; no real connection happens during generate.
- **Example**: `DATABASE_URL='postgresql://prisma-generate-stub:stub@localhost:5432/stub' npx prisma generate`

### Verifier script shape (DB-free flavour)
- **Where**: `scripts/verify-stock-analysis.ts:1-20` (canonical), `scripts/verify-warehouse.ts:23` (import-path convention)
- **Pattern**: `import "dotenv/config"`, local `assert(cond, msg)` that `console.error`s and `process.exit(1)`, single `async function main()` invoked at module bottom with `.catch(...)`. For pure-function verifiers, drop the `db` import / fixtures / cleanup.
- **Example**: see PRD US-002 technicalNotes for the skeleton

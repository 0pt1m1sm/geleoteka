## Big things to know

### 1. CMS now actually works

Two bugs fixed end-to-end:

1. `/contacts` was 100% hardcoded — no consumer of the CMSBlock table existed on the public side. Wired phone/email/address/hours from `contacts.*` keys. Same for the public layout's footer (visible on every public page).
2. `updateCMSBlock` server action never called `revalidatePath`, so pages would have stayed stale even if they did read from CMS. Now calls `revalidatePath("/", "layout")` which invalidates the whole public layout subtree.

New helper: `lib/cms.ts` exports `getCMS(key, fallback?)` and `getCMSMany(keys, fallbacks?)`. Cached per request via React `cache()`.

Affected public surfaces: `/contacts`, home page (stats + FAQ phone), public layout footer, `/parts/[slug]` (pickup address), `/rentals/[id]` (pickup address).

Still hardcoded (deferred): `lib/sms.ts` SMS templates. Touching SMS is sensitive (char limits, regional carriers).

### 2. Vehicle catalog is now data-driven, not hardcoded

The static `MODELS` array in `lib/models-data.ts` is gone. Catalog lives in three new tables: `Manufacturer`, `VehicleModel`, `VehicleGeneration`. Migration `20260505065939_add_vehicle_catalog` applied to local + prod. Comprehensive Mercedes-Benz lineup seeded: 1 manufacturer, 22 models, 63 generations (was 8 models / 14 generations in the static array).

Admin UI at `/admin/models` lets non-developers add/edit models + generations. Server actions in `app/actions/vehicle-catalog.ts`. Admin nav got a new "Модели и поколения" link under "Сайт".

Lib reorg:
- `lib/vehicle-catalog-types.ts` — pure types + label helpers, safe for client components
- `lib/vehicle-catalog.ts` — server-only queries (uses `import "server-only"` to enforce at build time)
- `lib/models-data.ts` — thin shim re-exporting only the client-safe pieces; partially-dead now

Year-range labels rendered as "1990–2018 · W463" / "2018–н.в. · W464" via `generationLabel()`. Visible in MyCarPicker, Step1ServiceVehicle chassis helper, /models, /models/[slug].

Customer can now pick any car in the lineup, even if we don't have parts indexed for it (that was a prior friction point).

### 3. Hero, FAQ, models page polish

- Hero desktop layout was broken (both halves stacked vertically at md+). Root cause: `.hero-divider-mobile { display: flex }` in globals.css lost source-order priority to Tailwind's `md:hidden`. Fixed via `@media (min-width: 768px) { .hero-divider-mobile { display: none } }` in the same stylesheet.
- FAQ accordion replaced its no-animation toggle with the modern `grid-template-rows: 0fr → 1fr` trick. Smooth 280ms collapse/expand.
- Models page: dropped the `priceNote` field (it surfaced things like "+30–50% AMG-наценка" — read as "we overcharge you"). Replaced with cleaner navigation + new "Запчасти для X" deep-link from the model detail page.

## Big things to know

### 1. G-Class W464 + W465 generations now exist in the catalog

The catalog had a stale `W464` entry that meant "2018–present luxury G". Wikipedia and aftermarket conventions now treat that as **W463A**. Renamed and added the two real new generations:

- **W464** (2022+) — utility/military line that replaced W461. Solid axles, three diff locks, 24V electrical, Euro III diesel. Sold via Mercedes Defense as Station Wagon (BA06) and Chassis Cab (BA09).
- **W465** (2024+) — facelift second-generation luxury G. Mild-hybrid I6 lineup, V8 retained for AMG, **G 580 with EQ Technology** (4-motor electric, 116 kWh, 587 hp).

W464 catch-up was done in this session (it had no curated trims after the rename; sourced from Mercedes GGB + ultimatespecs cross-check since auto-data has no W464 page).

### 2. Vehicle trims fully populated for all 22 Mercedes models

Started the session with **126 curated trims** across 14 generations (mostly G-Class + 4 hand-picked sets flagged "needs re-verification"). Ended with **675 curated trims across all 67 generations**. Average 10.1 trims/gen.

Source policy: every `CuratedGeneration` block in `prisma/seed-trims.ts` cites a single canonical URL. Auto-data.net is the primary source (per-generation pages); Wikipedia for engine codes when auto-data is silent on them; Mercedes GGB for W464; ultimatespecs as a documented secondary cite for W464 HP only.

Filters held throughout: zero `code LIKE '%USA%'` rows, zero China-only rows in prod (E 260 W214 was caught by the reviewer and removed; CLS 550 C219 same).

### 3. PHEV mapping convention

Schema enum is `PETROL | DIESEL | ELECTRIC | HYBRID` — no `DIESEL_HYBRID`. Decided once: **all PHEVs use `fuelType: "HYBRID"`**, base fuel + battery capacity in `notes`. Examples:
- `E 300 e` — `notes: "бензин+электро PHEV; батарея 25.4 кВт·ч; 2023–"`
- `GLE 350 de` — `notes: "дизель+электро PHEV; батарея 31.2 кВт·ч; 2019–"`
- 48V mild-hybrid stays `PETROL`/`DIESEL` (e.g. M 256 EQ Boost). Mild hybrid noted in `notes`, not in `fuelType`.

44 HYBRID rows in prod after this session. 38 ELECTRIC.

### 4. Engine code format normalised

All engine codes in `prisma/seed-trims.ts` now follow Mercedes-with-spaces format: `M 177`, `OM 656`, `OM 642 DE 30 LA`. Existing entries that had `M177`/`OM656` (no space) were converted in commit `992df3c` to keep the picker label uniform (the `trimLabel()` helper renders `engineCode` verbatim into the dropdown text).

### 5. Snapshot-reconcile pattern saved downstream data

Re-verified four generations (W205, W213, V167, X167) by deleting their curated trims and re-seeding from auto-data. The bare DELETE would have cascaded into `PartTrim` (CASCADE) and nulled `RepairOrder.trimId` (SET NULL). Mitigated with **snapshot-then-reconcile**:

1. Snapshot affected `PartTrim` rows + `RepairOrder.trimId` to a temp table BEFORE the DELETE.
2. Pre-delete + reseed.
3. Match snapshot rows by trim `code` to new trim ids.
4. Re-INSERT PartTrim links and re-SET RepairOrder.trimId.
5. Print explicit reconciliation report.

Final prod result: **4 PartTrim links preserved, 0 lost, 0 RepairOrder rows affected** (none existed). The pattern is documented in the plan's Task 7 verify block — reusable for any future re-verification.

### 6. New tooling: `verify-vehicle-trims.ts`

`scripts/verify-vehicle-trims.ts` mirrors the existing `verify-vehicle-catalog.ts`. Reports total gens, total curated trims, average per gen, and **fails (exit 1) on any generation with 0 curated trims**. Supports `--allow-bare <model-slug>:<chassis>` for intentionally-bare gens (none currently used).

Wired as `npm run verify-vehicle-trims`. Use case: run after editing `seed-trims.ts` to confirm coverage before pushing.

## Useful one-liners

```bash
# Verify catalog coverage (local or prod)
DATABASE_URL="<url>" npx tsx scripts/verify-vehicle-trims.ts

# Coverage breakdown per model
psql "<url>" -c "
  SELECT m.name, COUNT(g.id) AS gens,
    SUM(c.cnt) AS curated_trims
  FROM \"VehicleModel\" m
  JOIN \"VehicleGeneration\" g ON g.\"modelId\" = m.id
  CROSS JOIN LATERAL (
    SELECT COUNT(*) FILTER (WHERE NOT t.\"isDefault\") AS cnt
    FROM \"VehicleTrim\" t WHERE t.\"generationId\" = g.id
  ) c
  GROUP BY m.name ORDER BY m.name;"

# Re-seed prod (idempotent — upserts; never deletes)

  npx prisma db seed
```

# Платформа автосервиса Mercedes-Benz

Created: 2026-04-10
Category: Feature
Status: Draft
Research: Deep

## Problem Statement

**Проблема**: Ни один существующий игрок на рынке (AutoLeap, Shopmonkey, российские Splus, amggarage) не совмещает премиальный публичный сайт с полноценным личным кабинетом клиента и внутренней системой управления сервисом в одной платформе. Клиенты вынуждены записываться по телефону, не понимают, на каком этапе ремонт, и не получают прозрачных смет. Владельцы сервиса работают в Excel и WhatsApp.

**Для кого**: Специализированный автосервис Mercedes-Benz (Москва/Россия). Premium-сегмент. Клиенты — владельцы Mercedes, готовые платить за качество и комфорт.

**Почему сейчас**: Рынок дозрел до "Яндекс.Такси" уровня сервиса в автосервисе. Конкуренты либо устарели, либо закрывают только один слой.

## Core User Flows

### Flow 1: Запись на сервис (клиент)
1. Клиент заходит на сайт → выбирает услугу(и)
2. Вводит данные авто (модель / год / VIN или вручную)
3. Выбирает дату и время из реального календаря (не "оставьте номер")
4. Указывает контакты, примечания, загружает фото (опц.)
5. Получает подтверждение по SMS/email
6. Получает push-уведомление за день и в день визита

### Flow 2: Отслеживание статуса (клиент)
1. Клиент заходит в личный кабинет → видит статус своего авто
2. Статусы: Записан → Принят → Диагностика → В ремонте → Контроль качества → Готов
3. При каждой смене статуса — push/SMS
4. Видит ориентировочное время готовности
5. Читает заметки мастера в ленте

### Flow 3: Согласование сметы (клиент)
1. Мастер составляет смету в упрощённой панели
2. Клиент получает ссылку на согласование (SMS/email)
3. Видит каждую позицию: работы + запчасти
4. Может одобрить / отклонить отдельные пункты
5. Получает итоговый счёт (оплата пока офлайн)

### Flow 4: Программа лояльности (клиент)
1. После каждого визита — баллы на счёт (1 балл = 1₽ или % от чека)
2. Видит баланс в личном кабинете
3. Уровни: Серебро → Золото → AMG Club (больше баллов за визит)
4. Реферальная ссылка — друг записывается, оба получают бонус

### Flow 5: Управление записями (менеджер)
1. Менеджер заходит в упрощённую панель
2. Видит список записей на день / неделю
3. Календарь с цветовой кодировкой (тип работы, статус)
4. Меняет статус записи (Записан → Принят и т.д.)
5. Видит карточку клиента: авто, история, баланс лояльности

## Scope

### In Scope (Фаза 1: Публичный сайт + ЛК + Упрощённая админка)

**Публичный сайт:**
- Главная страница (hero, статистика, услуги, модели, отзывы, FAQ, карта)
- Страницы услуг (отдельная на каждую)
- Страницы моделей (C, E, G, GLE, S, AMG, EQ)
- Онлайн-запись (5-шаговый wizard с реальным календарём)
- Страница "О нас / Команда"
- Страница контактов
- Блог (статьи, советы, обзоры моделей)
- Мультиязычность RU/EN (RU первично, EN готовится)

**Личный кабинет клиента:**
- Регистрация / вход (email + телефон, сброс пароля)
- Мои автомобили (VIN-декодер, добавление вручную, индикаторы ТО)
- История обслуживания (по каждому авто)
- Отслеживание статуса ремонта в реальном времени
- Согласование смет (просмотр, одобрение/отклонение позиций)
- Баланс и история баллов лояльности
- Реферальная программа
- Уведомления (push/SMS/email)
- Просмотр предстоящих записей

**Упрощённая панель менеджера:**
- Дашборд: записи на сегодня, активные, выручка за день
- Календарь записей (день / неделя)
- Управление статусом записей
- Просмотр клиентов и их карточек
- Создание смет и отправка клиенту на согласование

**Технический стек (Фаза 1):**
- Frontend: Next.js 15 (App Router, SSR)
- Backend: Next.js API Routes + Server Actions
- Database: PostgreSQL + Prisma
- Auth: NextAuth.js (credentials + SMS/email)
- Storage: Yandex Object Storage (S3-compatible)
- Maps: Яндекс Карты API
- SMS: SMSC.ru
- Email: SendGrid / Postmark

### Explicitly Out of Scope
- **Онлайн-оплата (ЮKassa)** — Фаза 2 (счёт + ручной расчёт пока)
- **Магазин запчастей** — отдельный этап
- **Публикация на Avito/Drom/Zzap/Юла** — отдельный этап
- **Канбан заказ-нарядов** — упрощённое управление, полный канбан в Фазе 3
- **Цифровой осмотр авто (ЦОА)** с фото — Фаза 3
- **CRM клиентов** с тегами и LTV — Фаза 3
- **Управление складом запчастей** — Фаза 3
- **Полная аналитика** — Фаза 3
- **CMS для всех блоков сайта** — базовое управление контентом в Фазе 1, полное в Фазе 3
- **Английская версия** — после запуска RU

## Technical Context

### Архитектура: One Service (Not Multi-Tenant)
Это одна точка (один сервис), не SaaS-платформа. Данные только этого сервиса. Архитектура:
- Path-based routing ( `/` — публичный, `/cabinet` — клиент, `/admin` — менеджер)
- PostgreSQL RLS не требуется (один тенант)
- Все данные в одной БД, схема Prisma

### Real-Time Status Updates
- SSE (Server-Sent Events) для трекинга статуса в ЛК клиента
- Fallback: React Query polling каждые 30 секунд

### Authentication
- Credentials (email/password) + phone (SMS code)
- JWT + refresh tokens
- 2FA опционально (можно добавить позже)

### Календарь записей
- Библиотека: `@/calendar` or custom with date-fns
- Хранение: слоты в БД (time, master_id, duration, status)
- Блокировка двойной записи: unique constraint (time, master_id)

## Key Decisions

| Decision | Choice | Why |
|----------|--------|-----|
| Архитектура | One service | Быстрый старт, ниша = один сервис, не SaaS |
| Phase 1 scope | Публичный сайт + ЛК + упрощённый менеджер | Запускаемся с core, админка полная позже |
| Онлайн-оплата | Отложена | Счёт + ручной расчёт на старте, быстрее запуск |
| Программа лояльности | В Фазе 1 | Дифференцирующий фактор сразу |
| Магазин запчастей | Отложен | Сначала ядро, потом монетизация |
| Дизайн | Свой с нуля | Premium brand, не generic SaaS look |
| Язык | RU first | Контент для рынка, EN добавим после запуска |
| Stacked | Next.js + PostgreSQL | SSR для SEO + SPA для портала, зрелая экосистема |
| Integrations | Avito/Drom/Zzap отложены | Синхронизация с Splus/существующим сайтом пока не требуется |

## Research Findings

### Competitors Gap
Все конкуренты закрывают либо публичный сайт, либо личный кабинет, либо админку — но не всё вместе. amggarage.ru работает на WebAsyst WCMS — личный кабинет помечен "в разработке", платформа это просто не умеет. AutoLeap/Shopmonkey (США) — отличные SaaS, но без публичного бренд-сайта и не для российского рынка.

### UX Patterns (Industry Standard)
- **DVI (Digital Vehicle Inspection)**: чеклист с фото/видео, светофорный рейтинг, одобрение клиента. Стандарт индустрии (Tekmetric, AutoVitals, AutoLeap).
- **Real-time tracking**: статусы Записан → Принят → Диагностика → В ремонте → КК → Готов. Push/SMS при каждой смене.
- **Pre-fill**: returning customers видят своё авто pre-filled, не вводят повторно.
- **One-click calendar sync**: добавить в Google/Apple Calendar.

### Technical
- Next.js 15 App Router — SSR для SEO, SPA для портала.
- PostgreSQL + Prisma — зрелый стек, RLS доступен если понадобится multi-tenant.
- SSE для real-time (проще WebSocket, достаточно для status updates).
- ЮKassa — SDK есть, webhook, СБП + карты. Отложен на Фазу 2.
- SMSC.ru — HTTP API, ~1.5-2₽/SMS, Cyrillic.
- Avito/Drom/Юла/Zzap — XML-фид, один фид на все площадки. Отложено.

## Implementation Order (Inside Phase 1)

2. **Database schema**: users, cars, appointments, services, loyalty, estimates
3. **Auth**: registration, login, password reset (email/SMS)
4. **Public site core**: главная, услуги, модели, контакты
5. **Online booking wizard**: 5-step form + calendar slots + confirmation
6. **Client portal**: cars, history, status tracking, loyalty
7. **Simplified manager panel**: dashboard, calendar, appointment management
8. **Notifications**: SMSC.ru integration, push setup
9. **CMS basics**: editable text blocks, team profiles
10. **Blog**: articles, tags, author profiles
